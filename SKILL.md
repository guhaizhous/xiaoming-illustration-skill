# 文章插画 · Article Illustration

为中文内容生成极简线条正文配图。适用于文章观点、绘本内容、知识点拆解、教学场景。核心风格：白底、细线、蓝色点缀、小明作为动作主体、怪诞但清爽。

## 定位

不是商业插画，不是PPT信息图，不是可爱卡通。把文章里的判断、流程、结构、状态或隐喻变成一张清爽、怪诞、有创意、可读但不说明书的手绘解释图。

## 适用场景

| 场景 | 说明 | 示例 |
|------|------|------|
| 文章观点生图 | 为文章核心观点配图 | "知识复利"配一张小明在积累的金字塔上工作 |
| 绘本内容生图 | 为故事绘本生成场景插图 | 主角小明走进一片知识森林 |
| 知识点拆解生图 | 拆解复杂概念为可视化图 | 把"复利"拆解成时间×收益的物理隐喻 |
| 教学内容生图 | 教学场景、知识讲解配图 | 小明站在放大镜前观察知识晶体 |

## 风格规范

### 核心要素

- **线条**：细黑墨线，手绘微抖，不机械
- **背景**：纯白，不要纸纹、渐变、阴影
- **点缀色**：唯一蓝色 `#3B82F6`，落在最有意义的元素上
- **人物**：极小。物体宏大，人物渺小，戏剧性对比
- **留白**：大量留白，主体占画面40%-60%
- **标注**：少量蓝色/红色中文手写批注

### 小明 IP

小明是每张图的固定角色，不是装饰，而是正在认真参与系统运转的荒诞工作者。

**外形**：
- 黑色实心小怪物
- 蓝色圆点眼睛（与点缀色呼应）
- 细腿，偶尔有细胳膊
- 身体可以是黑豆、黑盒、漏斗、影子
- 表情空、呆、冷静、认真

**性格**：
- 很认真，但做的事有点荒诞
- 像一个低调的系统操作员
- 冷幽默，不卖萌
- 像在白板草图里真的负责某个工作

**常见动作**：
- 搬运素材堆积
- 拉线汇聚信息源
- 卡在断点里
- 操作"判断"杆
- 变成筛选漏斗
- 切开"素材"
- 盖章承接话术
- 牵着路径
- 举警告牌看坑
- 从洞里伸手但接不住
- 在旁边搬砖、搭桥、开门、分拣

**禁止**：
- 不要画成可爱吉祥物
- 不要给小明复杂服装、表情包
- 不要让小明只是站在旁边看
- 如果去掉小明，图的核心隐喻仍能成立，说明小明太装饰

## 工作流

### 第一步：分析内容

先读用户给的内容（文章、链接、笔记、截图），提炼：
- 核心观点是什么
- 哪些段落承担认知转折
- 哪些内容适合用图解释
- 哪些地方只适合文字不需要图

### 第二步：输出 Shot List

如果用户只是问"怎么配图"或"分析一下"，先给shot list。每张图写清楚：
- 放在哪段后
- 图的主题
- 核心意思
- 结构类型（小漫画/系统局部/前后对比/角色状态/概念隐喻/方法分层/地图路线）
- 小明在图里做什么
- 建议元素
- 建议中文标注词

**数量建议**：4-8张，短文1-3张，长文不超过9张。

### 第三步：生成图片

如果用户明确要求"生成/做图/输出"，用内置image_gen单独生成每张图。

**提示词必须包含**：
- 16:9横版中文正文配图
- 纯白背景
- 黑色手绘线稿，细线轻抖
- 唯一蓝色点缀 `#3B82F6`
- 小明作为核心动作主体
- 大量留白
- 少量蓝/红中文手写批注
- 禁止PPT、商业插画、幼稚可爱、复杂架构

**单张原则**：每张图只讲一个核心结构，不要把多张图拼在一张里。

### 第四步：检查与迭代

生成后检查：
- 小明是否只是装饰
- 画面是否太满
- 是否像流程图/PPT
- 中文批注是否太多或错字
- 背景是否干净白底
- 是否太可爱/幼稚/死板

有问题则重生成或局部编辑。

### 第五步：保存交付

保存到：`assets/<article-slug>-illustrations/`
命名：`01-topic-name.png`, `02-topic-name.png`

## 场景化模板

### 文章观点生图

```
用article-illustration为这篇文章生成4张观点配图。
要求：16:9横版、纯白背景、黑色手绘线稿、蓝色点缀、小明作为动作主体。
---
<粘贴文章>
```

### 绘本内容生图

```
用article-illustration为这个故事生成绘本插图，5张。
场景风格：怪诞但清爽，小明是主角。
---
<粘贴故事>
```

### 知识点拆解生图

```
用article-illustration把这个知识点拆解成3张图。
每张图只讲一个核心概念，用小明作为动作主体。

```

### 教学内容生图

```
用article-illustration为这个教学场景生成配图。
风格：简洁清爽、怪诞有创意、适合教学展示。
---
<粘贴教学内容>
```

## 构图类型参考

| 类型 | 适用场景 | 小明动作 |
|------|----------|----------|
| 小漫画 | 有前后对比、转折点 | 小明经历变化 |
| 系统局部 | 输入→处理→输出 | 小明在机器里操作 |
| 前后对比 | Before/After | 小明拉开关 |
| 角色状态 | 情绪/状态变化 | 小明表情/动作变化 |
| 概念隐喻 | 抽象概念可视化 | 小明与隐喻物互动 |
| 方法分层 | 层层递进 | 小明攀爬/挖掘 |
| 地图路线 | 路径选择 | 小明走路/迷路 |
| 分拣归类 | 信息整理 | 小明分类/搬运 |

## 判断标准

每张图必须满足：
1. 小明是核心动作主体，不是装饰
2. 画面有大量留白
3. 蓝色点缀只在最有意义的元素上
4. 中文批注少量且手写感
5. 极小人物，宏大物体，戏剧性对比
